# Contributing

Thanks for considering contributing to this roadmap.

## What we welcome

- Typos and clarity fixes
- Better explanations in notes
- New curated resources (with a short reason why they matter)
- Small, focused lab demos

## How to contribute

1. Fork the repo.
2. Create a branch for your change.
3. Make your changes (follow the templates and guidelines below).
4. Open a pull request with a short description of what you changed.

## Guidelines

1. **Follow the templates** — Use the same structure for `notes.md` and `resources.md` so the repo stays consistent.
2. **Keep English simple** — Short sentences, clear headings, explain jargon.
3. **Link formatting rule** — Use `[Title](url) — 1 line why it matters` for every resource link.
4. **One topic at a time** — Prefer one clear change per contribution.

## Before you push

- [ ] `notes.md` follows the template (see `templates/note-template.md`)
- [ ] `resources.md` is categorized with one-line reasons per link
- [ ] Images go in `assets/` with the naming convention: `<topic>-<type>-<short-desc>-v<number>.<ext>`
- [ ] Labs have a `labs/README.md` if the topic has demos
- [ ] Track `README.md` links are updated if you add or rename topics
